#Config File Formatting
The robot config file should be a plaintext file with the extension .cfg.
The format should be the following:
<Section Header>
<Section Type>
<Item name> <Item value>
<Item name> <Item value>
...

<Section Header>
...

Section Headers are used to identify a section to load.
Section header convention is all caps. Examples:
CONTROLS
MOTOR PORTS

Section Types are used to identify the type to interpret each item's value as.
Section type examples:
int
double

Items are used to parse what each value means. They are they Key for their value in the final Map.
Convention is lowercase and underscores. Examples:
button_x
y_axis

Each section should be separated by at least one empty line.
The empty line indicates the end of section.
